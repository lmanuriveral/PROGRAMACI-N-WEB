SERVIDORES WEB Y DE APLICACIONES

SERVIDORES WEB

Conocido a su vez como un servidor HTTP, programa informático que procesa una aplicación del lado del servidor,
el cual realiza conexiones con el cliente para ceder respuestas en cualquier lenguaje o aplicación por parte del cliente.

Existen varios tipos de servidores web basandose en las necesidades del usuario. Entre estos existen:

Según Windows:

  • Apache (XAMPP).
  • Nginx.
  • Servidor web IIS (Internet Information Services).
  • Lighttpd.
  • Caddy.
  • Easy PHP.

Según Linux:

  • Apache (LAMP).
  • Nginx.
  • Servidor web IIS (Internet Information Services).
  • Lighttpd.
  • Caddy.
  • Easy PHP.

Según MacOs:

  • Apache (MAMP).
  • Nginx.
  • Servidor web IIS (Internet Information Services).
  • Lighttpd.
  • Caddy.
  • Easy PHP.
  • Server2Go.

SERVIDORES DE APLICACIONES

Estos son conocimos como programas de servidor en una red distribuida con el fin de proporcionar correctamente un entorno de ejecucion para el programa de una aplicación.
Es decir, estos servidores vienen a ser el componente de tiempo de ejecución principal en todas las configuraciones y donde una aplicación se ejecuta realmente.

Existen varios tipos de servidores de aplicaciones basandose en las necesidades del usuario. Entre estos existen:

Según Java:

  • WidFly – JBoss Application Server.
  • GlassFish.
  • HP Bluestone.
  • Enterprise JavaBeans.
  • JOnAS.

Según Oracle:

  • Oracle WebLogic.
  • Oracle IAS.
  • Sun-Netscape Iplanet.

Según IBM:

  • WebSphere (IBM WebSphere).
  • IBM Domino.
  • Integrated web.

Según GNU:

  • Perl.
